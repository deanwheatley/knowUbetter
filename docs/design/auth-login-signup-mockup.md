## Notes

### Authentication Flow
- Email-based invitation detection (no codes needed)
- Display name required for all signups
- SSO pre-fills display name from provider
- Organization creation happens during Org Admin signup

### SSO Configuration
- Organization Admins control which SSO methods are enabled
- Disabled methods show tooltip with admin contact info
- knowUbetter (email/password) always enabled
- Future SSO (Okta, SAML) marked as "COMING SOON"

### User Profile Fields
- **Required at signup**: Email, Display Name, Password (if not SSO)
- **Auto-generated**: Avatar (can update later)
- **Optional**: About info (can add anytime)
- **Roles**: User, Team Admin, Org Admin, System Admin (hierarchical)

### Role Hierarchy
- System Admin > Organization Admin > Team Admin > User
- Higher roles inherit all lower role permissions
- Users select User or Org Admin at signup
- Team Admin assigned by Org Admin
- System Admin assigned manually
